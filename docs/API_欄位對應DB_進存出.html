G. 重點備註

  1. 🟧 計算欄位最多：aging(MIN(WMS_PayLoad.ReceivedDate))、Onhand(WMS_Inventory.Qty−配貨)、Cartons/Pallets(YAEP_Package 階層)、各 SUM、StorageStatus、SourceCons。DB 無現成欄位，後端 Transload 層算。
  2. 🟥 跨系統 T2：FreeDays、ServiceLines、Billing、Dashboard 未開票金額。WMS 拿不到，需 SPA 另呼或後端跨呼。
  3. 🆕 需新增 WMS 欄位：WMS_Manifest.SealNo/ContainerType/LoadingType、WMS_BOL.TrackingNumber/PODUrl、WMS_Vessel.StackableType、WMS_ShipMethod 補 LTL/FTL 兩筆。
  4. 客戶資訊鏈：WMS_Inventory 無 PartyUID → 一律經 WMS_Manifest_Item_List.ManifestUID → WMS_Manifest.PartyUID → YAEP_Party。
  5. Cartons/Pallets 換算全靠 YAEP_Package（階層 ParentUID + Quantity），DB 無「PcsPerCarton」單一欄位。
  6. D1 未定：ManifestName batch vs 每櫃 1 manifest，影響 ManifestName/containerCount/VesselUID 的對應。
